# Internationalization

All user-facing strings must use `react-i18next`. Import `useTranslation` with the appropriate namespace and use `t()` calls — never hardcode English text in JSX. This includes `aria-label` attributes.

**Namespaces** (files under `frontend/src/locales/en/`): `common`, `auth`, `dashboard`, `ritual`, `tasks`, `deferred`, `timeBlocking`, `admin`.

**No string concatenation** for sentences — word order varies by language. Use interpolation instead:

```jsx
// Wrong
`${count} tasks completed`

// Right
t('tasksCompleted', { count })  // en: "{{count}} tasks completed"
```

**Plurals** — use i18next plural suffixes, not inline ternaries:

```jsx
// Wrong
`${points} ${points === 1 ? 'point' : 'points'}`

// Right — en JSON has "points_one" and "points_other"
t('points', { count: points })
```

**Dates** — use `i18n.language`, never hardcoded `'en-US'`:

```jsx
new Date().toLocaleDateString(i18n.language, { ... })
```

See `docs/INTERNATIONALIZATION.md` for the phased rollout plan and full conventions.
