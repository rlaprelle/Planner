# i18n Phase 1: Infrastructure & Extraction

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers-extended-cc:subagent-driven-development (recommended) or superpowers-extended-cc:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Externalize every hardcoded user-facing string into `react-i18next` translation files so the app is i18n-ready while still rendering identically in English.

**Architecture:** Install `react-i18next` with namespace-based JSON translation files under `frontend/src/locales/en/`. Each JSX file imports `useTranslation` with its namespace and replaces hardcoded strings with `t()` calls. Shared strings (Cancel, Save, Loading) live in a `common` namespace referenced via `t('common:key')` prefix.

**Tech Stack:** react-i18next, i18next, i18next-browser-languagedetector

**Reference:** String audit at `docs/planning/user_design/I18N_AUDIT.md`

## Conversion Patterns Reference

These patterns appear repeatedly across all tasks. The implementer should internalize these before starting.

### Pattern A: Simple string
```jsx
// Before
<h1>Welcome back</h1>

// After
const { t } = useTranslation('auth');
<h1>{t('welcomeBack')}</h1>
```

### Pattern B: Ternary button state
```jsx
// Before
<button>{isSubmitting ? 'Saving…' : 'Save'}</button>

// After
<button>{isSubmitting ? t('common:saving') : t('common:save')}</button>
```

### Pattern C: Template literal with interpolation
```jsx
// Before
<p>{`${count} tasks completed`}</p>

// After
<p>{t('tasksCompleted', { count })}</p>
// JSON: "tasksCompleted": "{{count}} tasks completed"
```

### Pattern D: Ternary plural
```jsx
// Before
<span>{count} {count === 1 ? 'point' : 'points'}</span>

// After
<span>{t('points', { count })}</span>
// JSON: "points_one": "{{count}} point", "points_other": "{{count}} points"
```

### Pattern E: Object lookup table
```jsx
// Before
const WRAP_MESSAGES = {
  day: "That's a wrap for today.",
  week: "That's a wrap for the week.",
};
<p>{WRAP_MESSAGES[level]}</p>

// After
const WRAP_MESSAGES = {
  day: t('wrapDay'),
  week: t('wrapWeek'),
};
<p>{WRAP_MESSAGES[level]}</p>
```

### Pattern F: Interpolated aria-label
```jsx
// Before
<button aria-label={`Archive ${project.name}`}>

// After
<button aria-label={t('archiveProject', { name: project.name })}>
// JSON: "archiveProject": "Archive {{name}}"
```

### Pattern G: Cross-namespace reference
```jsx
// When a file's primary namespace is 'tasks' but needs shared strings:
const { t } = useTranslation('tasks');
<button>{t('common:cancel')}</button>  // references common namespace
<button>{t('archiveTask')}</button>    // references own namespace
```

```js
import common from './locales/en/common.json';

  .use(LanguageDetector)
  .use(initReactI18next)
  .init({
    resources: {
      en: {
        common,
        auth,
        dashboard,
        ritual,
        tasks,
        deferred,
        timeBlocking,
        admin,
      },
    },
    lng: 'en',
    fallbackLng: 'en',
    defaultNS: 'common',
    interpolation: {
      escapeValue: false, // React already escapes
    },
  });

export default i18n;
```
